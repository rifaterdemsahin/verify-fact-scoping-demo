## 🧭 TL;DR

A **Multi-Agent Research System** has a *synthesis agent* that frequently needs to verify claims while
combining findings. In the naive architecture **every** check bounces all the way back up to the
**coordinator**, which calls the **web search agent**, and then **re-invokes synthesis** with the result.
That adds **2–3 round trips per task** and **+40% latency** — even though **85%** of the verifications are
trivial fact-checks (dates, names, statistics).

The recommended answer is a classic **latency ↔ least-privilege** trade-off:

> ✅ Give the synthesis agent a **scoped, read-only `verify_fact` tool** for the **85%** of simple lookups,
> while the **15%** of complex, exploratory verifications keep delegating to the web search agent **through the coordinator**.

## 📋 The Exam Question & Answer

**Question — Multi-Agent Research System**

> During testing, you observe that the synthesis agent frequently needs to verify specific claims while combining findings. Currently, when verification is needed, the synthesis agent returns control to the coordinator, which invokes the web search agent, then re-invokes synthesis with results. This adds 2-3 round trips per task and increases latency by 40%. Your evaluation shows that 85% of these verifications are simple fact-checks (dates, names, statistics) while 15% require deeper investigation. What's the most effective approach to reduce overhead while maintaining system reliability?

**Recommended Architectural Solution**

> Give the synthesis agent a scoped `verify_fact` tool for simple lookups, while complex verifications continue delegating to the web search agent through the coordinator.
>
> *Why?* This represents a classic balance between system latency and agent least-privilege boundary separation. Giving the synthesis agent **full web access** compromises its narrow design scope and risks infinite loops or prompt dilution. However, forcing every trivial check to bounce all the way back up to the coordinator introduces severe latency. A fast, localized `verify_fact` lookup handles the 85% of minor checks directly, while preserving the coordinator path for heavy, exploratory research queries.

## 🏗️ Architecture & Why It's Superior

### Unidirectional dependency chain (no cycles)
```
domain.js  →  infrastructure.js  →  subagent-{naive,resilient}.js  →  coordinator.js  →  demo.js
```

### 🧩 Typed errors as architectural seams
Failures are **typed**, not stringly-typed. Callers branch programmatically on a `recoverable` flag:

```js
class TransientError extends AgentError { /* recoverable: true  → retry locally  */ }
class FatalError     extends AgentError { /* recoverable: false → escalate       */ }
```

### 🦆 One duck-typed contract — two behaviours
Both agents implement the *same* interface, so the **same coordinator** drives either:

```js
async synthesize(task, ctx) → { status, pending, findings, attemptLog }
```

### ❌ Naive synthesis agent (the anti-pattern)
No local capability — **every** trivial check surrenders control to the coordinator:

```js
async synthesize(task, ctx) {
  const next = task.claims.find(c => !answers.has(c.id));   // one unresolved claim
  if (!next) return { status: 'completed', findings };
  return { status: 'needs_verification', pending: [next] };  // ↩️ bounce up, every time
}
```

Each bounce costs **~1,900 input tokens** (250 coordinator overhead + 450 web-search prompt + 1,200 context replay) and a full round trip.

### ✅ Resilient synthesis agent (the answer)
A **scoped, bounded, read-only** `verify_fact` tool resolves the 85% locally; only the complex subset escalates:

```js
for (const claim of task.claims) {
  if (isSimpleFact(claim)) {
    const local = await this.verifyFact(claim);     // 🔍 70 tok, no round trip
    if (local.status === 'ok') { answers.set(claim.id, stored); continue; } // idempotent
  }
  pending.push(claim);                              // ↗️ escalate only the 15%
}
return pending.length
  ? { status: 'needs_verification', pending }       // 📦 structured escalation
  : { status: 'completed', findings };
```

### 🛡️ Structured escalation & loop safety
Escalation returns a **typed partial result** (`status`, `pending`, `findings`, `attemptLog`), never a thrown raw exception. The coordinator enforces a **bounded-retry circuit breaker** (`MAX_ROUNDS`) so a misbehaving agent can never trigger an infinite hand-off loop.

### ⚖️ Why not full web access? (the trap)
Giving the synthesis agent the *whole* web search capability breaks its narrow design scope → risk of **infinite loops** and **prompt dilution**. The scoped tool is intentionally **read-only and bounded** to the canonical fact store, so it *cannot* wander into exploratory research. Least privilege in action.
