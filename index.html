📋 The Exam Question & Answer

❓ Question

During testing, the synthesis agent frequently needs to verify claims while combining findings. Currently each verification returns control to the coordinator → invokes the web search agent → re-invokes synthesis with results. This adds 2–3 round trips per task and +40% latency. 85% of verifications are simple fact-checks (dates, names, statistics); 15% require deeper investigation. What reduces overhead while maintaining reliability?

✅ Recommended Answer

Give the synthesis agent a scoped verify_fact tool for simple lookups, while complex verifications keep delegating to the web search agent through the coordinator. Classic balance of latency vs. least-privilege boundary separation — a fast localized lookup handles the 85% trivial checks; the coordinator path is preserved for heavy exploratory research.
